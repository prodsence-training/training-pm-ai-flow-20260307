# 使用者訪談記錄

### 受訪者 Andy（後端工程師，3 年經驗）

「其實我每天就是把 Jira 上的 ticket 做完而已。
說真的，我不太確定這個 Epic 最後會變成什麼樣子，每次進來 Sprint 的項目總是又多又雜，也不知道跟產品的大方向有什麼關係。」

### 受訪者 Bob（前端工程師，1 年經驗）

「Sprint 有 10 天，前幾天都還好，感覺時間很充裕。
每天總是一堆插單插進來，因為緊急我就先做了。
我平常不太會去看 Jira 的 Burn-down，那個圖也不是那麼好找。
好像常常在第二周，像是第6天、第7天才發現慘了，要加班了！
如果每天能看到預期 vs 實際進展，我就不會這麼晚才發現。」

### 受訪者 Cindy（PM / PO）

「我覺得最痛苦的是，我很清楚產品現在往哪走、接下來要交付什麼，但團隊沒有同樣的畫面。每個 Sprint 總是做一堆非產品 Roadmap 上的項目，
當然，他們要插單都有問過我，雖然每次 Sprint 完成率看起來好像有進度，但 Roadmap 上的大功能進展感覺又遙遙無期，如果團隊每天都只看到自己的 ticket，我講再多願景跟策略，他們都覺得好像跟自己無關，如果能讓團隊一眼看到：現在做的功能是 Roadmap 裡的一部分，我就不用每次重新講願景。」

### 受訪者 Darcy（Scrum Master / Tech Lead）

「我每天主持 Standup，但坦白講，大家報的東西都很表面：
做什麼、昨天做了什麼、今天要做什麼。
大家每天都有一堆事要做，沒有人提阻礙。
我知道常常有維運的插單加進來，不過也不知道究竟佔了大家多少時間？
有時候等到第 8 天才發現 API 卡住了、UI 和後端不一致。
站會好像變成儀式，但沒有真的讓大家『看到風險』。
如果站會能看到哪些 Ticket 停滯，
如果插單能有優先順序，讓我們判斷是否值得現在插進來，
團隊交付的流程會比較順利一點」

### 受訪者 Emily（老闆）

「我常常不知道團隊有什麼問題？我每次問 PM，PM 都說『快好了』，但我看不出來哪裡快了。
PM 的產品 Roadmap 每一季都畫得差不多，我不確定這一季到底做了什麼？
Jira 的進度也藏在好幾層報表裡，不太直覺。
如果能把『目前 Epic 進度到哪、哪些 Story 風險最大』跟我報告，我可以協助他們釐清專注點。」
