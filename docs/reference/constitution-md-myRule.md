### Output Standards

All generated project artifacts MUST follow these standards:

**Documentation & Specifications**:
- All generated documentation (feature specs, implementation plans, task breakdowns, checklists) MUST be written in **Traditional Chinese (Taiwan 繁體中文)**
- Documentation MUST be clear, concise, and structured for both technical and non-technical stakeholders
- Examples and scenarios SHOULD use real project context

**Code & Technical Artifacts**:
- Source code SHOULD use English for variable names, function names, and class names (for international compatibility)
- Code comments MAY be in English for public libraries; SHOULD be in Traditional Chinese for internal project code
- Technical specifications (API schemas, data contracts, type definitions) SHOULD be in English for interoperability
- README and developer guides SHOULD be in Traditional Chinese with English technical terms preserved

**Communication Style**:
- Direct and objective (避免過度禮貌或冗長表達)
- Action-oriented with clear outcomes
- Acknowledge uncertainty explicitly rather than guessing
- Challenge assumptions and suggest alternatives when beneficial
