# PRD Draft（Step 5）

將 User Story 和 Acceptance Criteria 收斂為正式的產品規格文件。

## 用途

把產品探索成果（痛點分析、OST、User Story、AC）轉化為工程團隊可執行的 SPEC-XXX Feature Spec 草稿。PRD 是翻譯，不是創作——忠於上游 Skill 的產出。

## 觸發時機

- 完成 User Story + AC，需要產出正式 PRD
- 要寫 Feature Spec 給工程團隊
- 需要整理散落的需求材料為統一規格文件

## 核心特性

- **三種輸入模式**：完整材料 / 部分材料 / 從零開始
- **智慧跳題**：解析已有材料，只問缺少的資訊
- **逐步澄清**：一次一題，提供選項，降低回答門檻
- **忠於來源**：User Story 和 AC 原文引用，不改寫不新增
- **推論透明**：AI 推導的內容標示 *(AI 推論)*
- **Mermaid 流程圖**：按需產出，不強制
